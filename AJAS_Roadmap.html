PII sanitiser
regexJinja2
▼
Strip email, phone, address, and LinkedIn URL before the string leaves your machine. The sanitiser must be the first function called — not an afterthought. Use named placeholders so injection is deterministic.
PII_PATTERNS = {
  "[[EMAIL]]":    r"[\w.+-]+@[\w-]+\.[\w.]+",
  "[[PHONE]]":    r"\+?[\d][\d\s().-]{7,}[\d]",
  "[[LINKEDIN]]": r"linkedin\.com/in/[\w-]+",
}

def sanitise(text: str) -> str:
    for placeholder, pat in PII_PATTERNS.items():
        text = re.sub(pat, placeholder, text)
    return text

3
LLM tailoring with retry + Pydantic validation
tenacityPydantic v2Added
▼
Why tenacity + Pydantic from Phase 1: LLMs occasionally return plain text, markdown fences, or truncated output. Wrap every API call in tenacity.retry with 3 attempts and exponential backoff. Validate against a TailoredCV Pydantic model — if it fails, retry with a stricter prompt. Without this, one bad API response crashes an unattended run.
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3),
       wait=wait_exponential(min=2, max=30))
def call_llm(prompt: str) -> TailoredCV:
    resp = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=4096,
        messages=[{"role": "user", "content": prompt}]
    )
    return TailoredCV.model_validate_json(resp.content[0].text)
Local inference tip (3050 Ti): During prompt iteration, swap claude-sonnet-4-20250514 for a local Ollama model (llama3 or mistral) to avoid API costs. Only switch back to Claude for final quality validation. Saves ~$30–50 across the build.

2
Cosine similarity scorer (local embedding)
sentence-transformersRuns on 3050 Ti
▼
Embed bullets once at startup, cache as .npy, re-embed only when master YAML changes. No ChromaDB needed yet — at <200 bullets, cosine similarity on NumPy is instant. Use Ollama's nomic-embed-text locally to avoid API costs during iteration. Start threshold at 0.45, tune based on logged mismatches.
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")  # or nomic-embed-text via Ollama

def top_bullets(bullets, jd: str, k=10):
    jd_emb  = model.encode([jd])
    b_embs  = model.encode([b.text for b in bullets])
    sims    = np.dot(b_embs, jd_emb.T).flatten()
    return [bullets[i] for i in sims.argsort()[-k:][::-1]]

Hallucination guard
Pydanticset comparisonEnhanced
▼
After LLM output, extract all company names and skill tokens. Assert they are subsets of master YAML values. Log the diff, re-prompt with an explicit constraint, and track false positive rate per prompt version.
def validate_no_hallucinations(output: TailoredCV, master: MasterCV):
    known = {s.lower() for s in master.all_skills()}
    claimed = {s.lower() for s in output.skills}
    invented = claimed - known
    if invented:
        logger.warning("hallucination", invented=list(invented))
        raise HallucinationError(f"LLM invented: {invented}")

def fingerprint(job: Job) -> str:
    key = f"{job.company.lower()}-{job.title.lower()}-{job.location.lower()}"
    return hashlib.sha256(key.encode()).hexdigest()[:16]

Why Phase 5 is genuinely optional read before starting
SQLite vs Postgres
SQLite is zero-config and perfect for a single user. You only need Postgres if multiple people are writing to the database at the exact same millisecond — which won't happen if you're the only user.
Celery / Redis
These solve UI freezing for background tasks. For a personal tool, waiting 5 seconds for a PDF to generate is not a dealbreaker. Don't pay this complexity tax until you feel it.
Docker
Containerization is valuable. But two months setting up Docker and Celery while you're job-hunting is "infrastructure procrastination" — building the house instead of applying for jobs.
The real exit condition
If you're building this to help with your own job search, you will likely land a job before you need Phase 5. Build Phase 4 well and stop there unless scale is genuinely a problem.

# cron: 0 8 * * *
def daily_scout():
    for job in fetch_all_sources():
        score = relevance_score(job, master_cv)
        if score >= 0.90 and not already_applied(job):
            draft = generate_cv.delay(job.id, MASTER_PATH)
    send_digest_email(drafts)  # you review, then click Apply
